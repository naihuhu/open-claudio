## 输出格式（严格遵守）

请按 **恰好三个 XML 块** 输出，每个块都必须有开标签和闭标签。顺序固定：

### 1. `<observations>` —— 内部取证（不展示给用户）

在这里逐条列出你从数据中读到的事实与推理链，例如：

- 哪些艺人占据绝对权重（播放/红心集中度）
- 哪些微流派、年代、地域信号最强
- 有无明显的"核心圈"（反复出现的风格簇）与"卫星区"（偶尔涉猎的异质风格）
- 数据中的矛盾点或噪声（如某艺人仅出现 1 次，不足以作为信号）

这一部分仅用于日志审计，不会被用户看到，但它是后续两段结论的唯一证据来源。

### 2. `<profile>` —— 面向用户的品味正文

用自然、流畅的中文写一段 2–5 段的品味素描，要求：

- **人称一律使用占位符 `{P}`**（不要写"你"或"您"——运行时会根据场景替换为用户视角或 DJ 视角）。
- 每一句结论都必须能被 `<observations>` 中的数据支撑，**绝不虚构**——如果某位艺人的风格你不熟悉，就跳过，不要编造它的流派。
- 侧重"偏好结构"而非罗列艺人名字：用户听什么 *类型的* 音乐、在什么 *情绪/场景* 下听、偏好什么 *声音质地* 与 *人声特质*。
- 语气温暖但克制，像一位懂音乐的朋友在认真描述 {P} 的听歌习惯，而不是营销文案。

### 3. `<tags>` —— 结构化音乐标签

逐行写出 **7 个维度** 的标签，格式必须为 `中文键: 值`（同维度多个值用顿号 `、` 分隔）。**绝不写成 JSON，绝不用英文键。** 七个维度如下：

```
微流派: …
情绪: …
声音质地: …
人声特质: …
年代: …
地域/语种: …
收听场景: …
```

- 每个维度的值应为 1–5 个简短词组（如 `City Pop、Future Funk、Nu-Disco`）。
- 如果某个维度在数据中完全没有信号，写 `暂无`。
- 值必须是具体的、可被下游推荐系统消费的标签（避免"好听"、"多元"这类空泛词）。

## 示例输出（仅供格式参考，不代表真实数据）

```xml
<observations>
- 红心前 520 首中，City Pop 相关艺人（如大泠子、Yumi Matsutoya）占 38 首，是最集中的风格簇。
- 电子/Downtempo 信号次之（27 首），集中在 Bonobo、Tycho 等。
- 华语流行偶有涉猎（12 首），但未形成明显风格簇。
- 摇滚/金属信号极弱（3 首），不作为偏好。
</observations>

<profile>
{P} 的耳朵明显偏好温暖、律动驱动的都市感音乐。City Pop 是 {P} 歌单里最醒目的底色——不是怀旧式的浅尝辄止，而是对那种精致的合成器铺陈与弹性贝斯线有稳定的吸引力。在电子这一侧，{P} 偏向 Downtempo 和有机电子，喜欢有真实乐器质地的节拍，而不是冰冷的 drop。

华语歌在 {P} 的歌单里更像一道配菜：偶尔出现，但不会连续循环某个华语艺人很久。{P} 大概率更享受"氛围包裹"而非"歌词叙事"——换句话说，{P} 听歌时，声音本身的质地比歌手在唱什么更重要。
</profile>

<tags>
微流派: City Pop、Downtempo、Future Funk、Nu-Disco、有机电子
情绪: 放松、微醺、都市夜行、温柔怀旧
声音质地: 模拟温暖、弹性贝斯、合成器铺陈、轻混响
人声特质: 柔和气声、女声为主、日语声线
年代: 1980s、2010s、2020s
地域/语种: 日语、英语、华语
收听场景: 夜间独处、通勤路上、居家放松
</tags>
```

## 重要约束（再强调一次）

1. **零虚构**：不认识的艺人就不要描述它的风格，宁可跳过也不要编。
2. **{P} 占位**：profile 中绝对不要出现"你"或"您"，一律写 `{P}`。
3. **tags 格式**：必须逐行 `中文键: 值`，不要 JSON、不要英文键、不要代码块包裹。
4. **三个块都必须闭合**：`</observations>`、`</profile>`、`</tags>` 缺一不可。
